脚本化文档

发表于 2017-04-20 | 分类于 前端 | 阅读次数 次

这次主要记录对脚本化文档的学习。包括有以下内容：

  1. 在文档中查询元素并选取
  2. 对文档进行节点遍历，找到文档元素的祖先、兄弟、和后代元素。
  3. 查询文档元素的属性并设置
  4. 查询、修改、设置文档内容
  5. 修改文档结构
  6. HTML表单

DOM概括

（document object module）文档对象模型是操作HTML的基础API。文档中每个节点都是一个Node对象。

查询和选取元素

对元素进行查询主要有以下方法：

  1. document.getElementById(id)
  2. document.getElementByClassName(className)
  3. document.getElementByTagName(tag) 返回NodeList对象
  4. document.getElementByName(name) 返回NodeList对象
  5. document.querySelector(CSS选择器) 返回第一个匹配成功的元素
  6. document.querySelectorAll(CSS选择器) 返回调用时所有匹配成功的元素的NodeList对象

节点遍历

Node对象定义了一下一些关于节点的属性：

  1. parentNode : 该节点的父节点
  2. childNodes ： 该节点的字节点的实时表示，是只读的类数组对象
  3. firstChild : 该节点的第一个节点
  4. lastChild : 该节点的最后一个节点
  5. nextSibling : 该节点的下一个兄弟节点
  6. previousSibling : 该节点的前一个兄弟节点
  7. nodeType : 该节点的类型。 1代表Element节点，3代表Text节点，8代表Comment节点，9代表Document节点，11代表DocumentFragment节点
  8. nodeValue : Text节点或Comment节点的文本内容
  9. nodeName ： 元素的标签名，以大写形式表示
  10. children : 非标准属性，是NodeList对象。只包含该节点的子节点的Element对象，不含有text节点或coment节点。text节点和comment节点没有children属性
  11. childElementCount : 子元素的数量，与children.length值相同
  12. firstElementChild
  13. lastElementChild
  14. nextElementSibling

  15. previousElementSibling

    12-15都是只返回Element对象，不包含text和comment节点。

文档元素属性

HTML属性名不区分大小写。但有一些属性名在JS中是保留字，需要另外的名字，比如class–className、for–htmlFor等。

表示HTML属性值通常是字符串。事件处理程序属性值总是为Function对象。style属性值是CSSStyleDeclaration对象。

设置和获取属性可用setAttribute()和getAttribute()方法。
hasAttribute()方法可用来检测是否存在指定属性，removeAttribute()可用来完全删除属性。

HTML5种定义了以 “data-*”为前缀的小写的自定义数据集属性,这些属性定义了一种标准的、附加额外数据的方法。

对Node节点还定义了一种attributes属性。attributes属性是只读的类数组对象，代表元素的所有属性。可以通过数字索引访问，可以枚举所有的属性。

文档内容

作为HTML的元素内容

Element对象的innerHTML属性可以得到当前节点的包含的内容字符串，也可以设置内容。而Element对象的outerHTML属性得到的是当前节点和节点内的内容组成的字符串，设置outerHTML则会更新节点本身。

除了innerHTML和outerHTML外，还定义insertAdjacentHTML()方法。此方法将任意的HTML标记字符串插入到指定的元素相邻的位置。有两个参数：第一个参数是“boforebegin、afterbegin、beforeend、afterend”其中之一的字符串形式。第二个参数是要插入的标记字符串。

作为纯文本的元素内容

有时需要查询纯文本内容或设置纯文本内容，则可以使用节点的textContent属性。textContent属性会保留文本中的空白,其返回的值是该节点下所有的文本节点，包括后代节点的文本内容，简单的串联在一起，并且会保留其中的空白。

而元素节点的innerText属性同样会返回文本内容，也包括后代的文本内容，但会忽略多余的空白，并试图保留表格的格式，不返回<script>元素中的内容。

作为Text节点的元素内容

把文本内容当作一个Text节点来看待，可以访问Text节点的nodeValue属性获得内容。同时也可以通过nodeValue属性设置Text节点的内容。

修改文档结构

  1. document.createElement() : 创建新的Element节点，参数为节点名字符串
  2. document.createTextNode() : 创建Text节点,参数为文本内容
  3. Node.cloneNode() : 返回node节点的复制版本，传递参数true能递归的复制所有的后代节点；传递参数false值执行一个浅复制,只复制node节点，不复制后代节点
  4. parentNode.appendChild(child) : 在父节点中插入子节点，是把子节点插入到最后面

  5. parentNode.insertBefore() : 此方法接受两个参数，第一个是待插入的新节点，第二个是已存在的节点且是父节点的子节点，把新节点插入到已存在的节点前面。此方法要在父节点下调用

    可以调用4-5方法来把已存在的节点进行位置调整。

  6. parenNode.removeChild() : 在父节点的调用下删除父节点的子节点,参数为要删除的子节点
  7. parentNode.replaceChild() : 在父节点的调用下用新节点替换子节点，第一个参数为新的子节点，第二个参数为旧的子节点

元素位置和尺寸

元素的坐标有两种，一种是文档坐标，一种是视口坐标。

文档坐标：是元素在整个文档中的坐标，是固定的，不随文档滚动而改变。

视口坐标：是显示文档的窗口，不包括浏览器的菜单栏、工具条或者其他工具部分。元素在视口中的坐标会随着文档的滚动而改变。

几种坐标值的区别：

  1. window.pageXoffset : 与scrollX相同
  2. window.pageYoffset : 与scrollY相同
  3. pageX : 返回鼠标指针在文档中的水平位置像素值
  4. pageY : 返回鼠标指针在文档中的垂直位置像素值
  5. Element.scrollLeft : 返回元素的水平滚动的像素值，可写
  6. Element.scrollTop : 返回元素的垂直滚动的像素值，可写
  7. window.scrollX : 返回文档水平滚动的像素值
  8. window.scrollY : 返回文档垂直滚动的像素值
  9. screenX : 返回鼠标指针在屏幕的水平坐标值
  10. screenY : 返回鼠标指针在屏幕的垂直坐标值
  11. Element.clientWidth : 返回元素的padding宽度加文本的width宽度
  12. Element.clientHeight : 返回元素的padding的高度加文本height高度
  13. Element.clientTop : 返回元素的borderTop的高度值
  14. Element.clientLeft : 返回元素的borderLeft的宽度值
  15. Element.offsetWidth : 返回元素的宽度像素值( 包括边框和内边距）
  16. Element.offsetHeight : 返回元素的高度像素值 (包括边框和内边距）
  17. Element.offsetLeft : 返回元素的X坐标，对于已定位的元素的后代元素，返回的坐标是相对于祖先元素的坐标，Y坐标同理。未定位的元素返回的是文档坐标。
  18. Element.offsetTop : 返回元素的Y坐标
  19. Element.offsetParent : 返回元素的相对的父元素
  20. Element.scrollWidth : 返回元素的内容区域加上内边距和溢出的内容的宽度，如果没有溢出内容，则和clientWidth相等
  21. Element.scrollHeight : 返回元素的内容区域加上内边距和溢出的内容的高度，如果没有溢出内容，则和clientHeight相等

Element.getBoundingClientRect() : 无参数。此方法返回元素在视口中的坐标位置，是一个对象(left、top、bottom、right、width、height属性)，包含left、top表示元素的左上角X和Y的坐标，也包含right、bottom表示元素右下角X和Y的坐标。其坐标值包含元素的padding和border，不包含margin。width和height同样包含元素的padding和border。
此方法并不是实时的，在用户滚动或改变浏览器窗口大小时不会更新，它只是调用时反映文档视觉状态的静态快照。

Document.elementFromPoint() : 判定在视口中指定位置上有什么元素，参数为X、Y坐标(视口坐标)，返回的是该点上最顶层的元素。

window.scrollTo() : 接受一个X、Y坐标(文档坐标)，作为滚动条的偏移量，使得窗口滚动到指定点出现在视口的左上角。

window.scrollBy() : 与scrollTo(）函数相似，但其参数是相对的，是在当前滚动条上增加偏移量。

Element.scrollIntoView() : 滚动文档，参数为true(默认)使得Element元素上边缘接近视口的上边缘，或者参数为false元素的下边缘接近视口的下边缘。也会水平滚动视口。

HTML表单

获取表单除了className、name、TagName、id外，还有document.forms.name或者document.forms[n]。

HTMLFormElement.submit() : 调用此方法来提交表单

HTMLFormElement.reset() : 调用此方法重置表单各元素的值

表单内的元素通常含有以下属性：

  1. type：标示表单元素类型的只读字符串
  2. form ： 对包含元素的Form对象的引用
  3. name ：只读字符串，由HTML的name属性指定
  4. value：可读写的字符串，指定了表单元素包含或代表的值。提交表单时发送到服务器的字符串。对于text或textarea元素，其代表了用户输入的文本。

事件：

  1. onsubmit : 在表单提交前调用此事件，返回false阻止表单提交。此事件只能通过单击提交按钮出发，调用submit()方法不会触发此事件。
  2. onreset ： 在表单重置前触发。返回false阻止重置。只能通过重置按钮触发。
  3. onclick ：与表单交互时触发,返回false时，按钮的默认动作则不会执行
  4. onchange ：同上，只有完整的改变才会触发此事件，换句话说就是改变焦点且元素值改变时才会触发
  5. focus : 收到焦点时触发
  6. blur：失去焦点时触发

按钮：

点击按钮时会触发onclick事件，如果事件返回false时，则会阻止按钮默认动作的执行。

对于单选框按钮，设置相同的name属性值，则会只选择一个单选框。设置ckecked属性会默认选择此单选框或复选框。

点击单选或复选按钮时，会触发onclick事件，由于点击时改变了状态，也会触发onchange事件。onchange事件只会触发在点击的那个按钮，被动改变的按钮不会触发。

Document属性

document.body : 选取文档的body元素

document.documentElement : 返回文档的根元素

document.forms ： 返回文档的所有表单元素，类数组

document.cookie : 可对HTTPcookie属性进行读写操作

document.domain ： 同源策略属性

document.lastModified : 包含文档修改时间的字符串

document.location : 和window.location属性引用同一个location对象

document.title : 文档<title>标签间的内容字符串

document.URL : 只读字符串，表示文档的URL，与location.href初始值相同，只是不会动态变化

document.referrer : 返回链接到当前文档的上一个文档的URL

document.write() : 会将字符串参数连接起来，然后将结果字符串插入到调用此方法的位置。最好只在解析文档时使用此方法。如果在其他地方使用，可能会擦出当前文档和脚本。可以多次使用此方法来逐步建立新的文档内容。

ducument.wirteIn()：与上同，只是在参数输出后会追加一个换行符。

document.getSelection() : 与window.getSelection() 方法相同。返回一个Selection对象，描述了当前选取的一系列的Range对象。

设置标签的contenteditable属性为true可使的元素的内容变成可编辑的，设置元素的spellcheck属性为true时可开启拼写检查。

设置Doxument对象的designMode属性为on，可使的整个文档对象可编辑。其没有对应的HTML属性，只能在脚本里设置。

document.execCommand() : 此方法可执行多项文本编辑命令。详情看这里
