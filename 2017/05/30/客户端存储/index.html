客户端存储

发表于 2017-05-30 | 分类于 JavaScript | 阅读次数 次

这次主要对客户端存储进行学习，了解在客户端对数据的处理和应用。

web存储

web存储中有两种方式存储，localStorage和sessionStorage，都是window对象下的属性。这两个属性都代表同一个Storage对象—-持久化关联数组，索引值和值都是字符串形式。两种方式的区别在于存储的有效期(能存储多长时间)和作用域(谁能访问)。

localStorage和sessionStorage具体区别如下:

  1. localStorage存储的数据是永久性的，一直存在电脑上，除非刻意删除；sessionStorage的数据是临时的，一旦窗口、标签页、浏览器关闭，sessionStorage存储的数据就会被删除。
  2. localStorage的作用域限定在文档源级别的。文档源由协议、端口、主机名确定。同源的文档间共享localStorage，可以互相修改、覆盖对方的数据。同时localStorage的作用域也受到浏览器的限制。即使文档源相同，不同浏览器不能访问统一站点的localStorage数据；与lcalStorage一样，sessionStorage的作用域也限定在文档源中。同时sessionStorage的作用域还限定在窗口中。即使访问的是相同的站点，不同标签页或窗口有不同的sessionStorage数据，不能共享数据。sessionStorage所限定的窗口指的是顶级窗口，如果含有内嵌的子窗口，其sessionStorage数据是可以共享的。

localStorage和sessionStorage的API:

  1. setItem() : 实现数据存储，传入属性名和属性值。localStorage.setItem('x',123)
  2. getItem() : 获取存储值，传入属性名
  3. removeItem() : 删除属性值，传入属性名
  4. clear() : 删除所有存储的数据
  5. key() : 传入一个0-length-1范围内的数值，返回对应的属性名

无论何时当localStorage或sessionsStorage存储的数据发生真正的改变时，会在对该数据可见的窗口或标签页上触发存储事件。但在数据发生改变的窗口上不会触发事件。比如在在两个有相同的文档源的窗口中，一个窗口的localStorage数据发生了改变，另一个窗口就会触发存储事件。
sessionStorage会在有关系的窗口中才会触发事件。可以通过onstorage事件监听存储事件。
localStorage和存储事件采用的是广播机制，浏览器会对目前正在访问相同站点的所有窗口发送消息。

cookie

cookie是在web浏览器端存储的少量数据，会自动在服务端和浏览器端进行数据传输，服务端可以读写web端的cookie数据。cookie数据的操作需要在一字符串的形式在window.document对象上的cookie属性来完成。window.document.cookie
要使用cookie之前，可以通过navigator.cookieEnbled属性检测cookie是否启用。

cookie有效期和作用域

cookie的默认有效期短，只能维持在浏览器的绘画期间，浏览器关闭cookie数据也会丢失。与sessionStorage的有效期区别在于cookie的作用不局限在某一窗口，cookie的有效期和整个浏览器的进程一致，而不是单个浏览器窗口的有效期一致。可以通过设置cookie的max-age属性来延长cookie的有效期。当设置有效期后，浏览器会创建一个存储cookie数据的文件，直到有效期结束删除文件。

cookie的作用域是通过文档源和文档路径确定，也可以通过cookie的path和domain属性配置。cookie与创建他的页面有关，同时也会对创建他的web页面的同目录页面或子目录页面可见。因此可以设置页面的路径(path属性)来使得整个网站都可以使用cookie数据。

cookie的path属性不能被用作访问其他cookie的控制机制。如果一个页面想要访问同一站点的其他页面的cookie，只需把其他页面以隐藏<iframe>的形式加载进来。在读取cookie就行。

设置cookie的domain属性，可以使子域间相互访问对方cookie数据。cookie的domain属性默认为当前web服务器的主机名。cookie的域只能设置为当前服务器的域。

cookie的secure属性表明cookie是以何种形式通过网络传递，布尔值。cookie默认是以不安全的形式传播(http协议)。如果cookie被标示为安全的，secure值为true，则只能通过https或其他安全协议传递。同时设置cookie的HttpOnly属性可以有效的防止程序访问cookie，避免xss攻击。

cookie数据的设置使用名值对name=value，其中值中不能使用分号、逗号、空白符。多个名值对使用分号和空格隔开。延长有效期使用max-age属性。
读取cookie数据时返回的是字符串，由多个名值对组成，每个名值对由分号和空格分隔。要使用相应的解码方式解码cookie数据。

cookie设计为了给服务端脚本存储少量的数据，在每次网络请求时都会发送cookie到服务器。因此cookie有所局限，每次cookie保存的数据不能超过4KB，浏览器为每个服务器存储的cookie数量不超过20个，浏览器保存的cookie数量不超过300个，但现代浏览器允许保存超过300个的cookie数量。
