window对象

发表于 2017-05-12 | 分类于 JavaScript | 阅读次数 次

这次主要学习window对象相关的方法和属性。了解window对象在编程中起到的作用和如何使用window对象的方法和属性。

计时器

window对象的计时器方法时setTimeout() 和 setInterval(),在指定时间后执行回调函数。与之相对应的有clearTimeout()和clearInterval()方法，取消计时器函数的执行。详细介绍看这里。

浏览器定位导航

window对象的location属性引用的是Location对象，表示的是浏览器窗口中当前显示文档的URL，并定义了方法来使窗口载入新的文档。

同时，window.location === document.location 的值为true，表示window和document引用的是同一个Location对象。

但是document中的URL属性与document.location.href属性有所不同。document.URL表示文档首次载入后保存的文档URL静态字符串。如果定位到文档中其他的片段标识符，Location对象会发生更新，而URL属性不变。

解析Location对象属性和方法：

href：值为文档的完整的URL文本字符串，改变其值会显示新的文档

protocol：值为URL的连接协议，包含逗号，比如 https:

host：值为URL的主机名，包含端口号

hostname：值为URL的主机名，不包含端口号

port：值为URL的端口号

pathname：值为URL中／后的文件地址字符串

search：值为URL中？后面的查询字符串参数,包括？，可以对URL进行参数化

hash：值为URl中#后面的片段标识符，包含#，改变其值会在当前文档中跳转的指定片段

assign(url)：可使浏览器窗口载入指定的URL地址的文档，参数为指定的url

replace(url)：使浏览器载入指定url的文档，但会把当前文档从浏览记录中删除，使用浏览器后退按钮则不会显示当前文档。

reload()：重新载入当前文档，参数为可选布尔值，为true，文档会始终从服务器重载，为false或默认则会从缓存中重载

toString()：返回完整的URL地址字符串。与href同义，但不能用此方法修改URL

history对象

window对象的history属性引用的是History对象，History对象用来把窗口的浏览历史用文档和文档状态列表的形式来表示。

History对象属性解析：

length：只读属性，返回浏览历史列表中元素的数量。但脚本不能访问浏览历史的URL

scrollRestoration：允许web应用在浏览历史导航中明确设置默认的滚动条恢复行为。其值可以是auto或manual

state：返回在浏览历史栈中顶部的状态值。这不需要popstate事件就可以查询状态。

back()：相当于浏览器的后退按钮，返回上一个文档页面，如果窗口包含子窗口，子窗口的历史记录会按顺序插入到主窗口的历史记录中

forword()：相当于浏览器的前进按钮，返回前一个页面

go()：接受一个数字参数，可以向前或向后跳转多个页面，history.go(-3)

pushState()：该方法会将当前文档页面的状态信息添加到浏览器历史记录中，如果一个web应用进入一个新的状态时，会自动调用此方法。第一个参数是一个状态对象，包含当前文档页面所有的信息；第二个参数是一个可选的标题字符串，浏览器可以使用它来表示浏览器历史记录中保存的状态；第三个参数是一个可选的URL，表示当前状态的位置。此方法也是HTML5管理浏览器历史记录的方法。

replaceState()：此方法与上面方法的参数相同，不同的是此方法不是把文档的新状态添加到历史记录中，而是用新的状态替代当前的历史状态。

管理历史记录出了pushState()方法外，还可以利用Location.hash和hashchange事件处理。记录Location.hash的片段标识符，当hash值发生改变时，会触发hashchange事件，解析hash值，则会载入状态信息改变的文档。

navigator对象和screen对象

window的navigator和screen属性分别引用的是Navigator对象和Screen对象。

Navigator对象展示了一些浏览器厂商和版本的信息，还包括一些地理位置、网络连接等杂项信息，Navigator对象详细属性解析看这里。

Screen对象提供了窗口显示的大小和可用颜色数量的信息。

对话框

alert()：项用户显示一条信息并等待用户关闭。

confirm()：显示一条信息，要求用户单击取消或确定，返回一个布尔值。

prompt()：项用户显示一个输入框等待用户输入，返回输入值。

以上的对话框内显示的文本都是纯文本。都会产生阻塞，也就是在关闭对话框前，代码会停止运行，文档会停止载入，直到对话框关闭才会运行代码或载入文档。

showModalDialog()：显示一个包含HTML格式的模态对话框，即显示出来则不可以点选位于其下面的对话框。其有三个参数，第一个参数是用以指定提供对话框HTML内容的URL；第二个参数是一个任意值，可通过window.dialogArguments属性访问。第三个参数是一个列表，可以配置对话框的尺寸或其他属性。此方法在窗口关闭前不会返回值，关闭后可以通过window.returnValue属性访问返回值。

错误处理

window.onerror是一个事件处理程序，检测到有异常时会调用此事件，并把错误信息输出到控制台。赋予此事件一个函数时，发生异常时就会调用此函数处理异常。函数有三个字符串参数，第一个是描述错误的一条信息，第二个参数是一个存放引发错误的JS代码所在的文档的URL字符串，第三个参数是文档中发生错误的行数。如果返回值是false，表示通知浏览器已经处理了错误，不需其他操作，不显示错误信息。但在firefox要返回true才行。

多窗口和窗体

每个可窗口或标签页都是独立的上下文，每个上下文中都有独立的window对象，且相互间无关联。

HTML可通过<iframe>标签创建内嵌子窗口文档，且其上下文由其自己的window对象表示。

对客户端JS来说，窗口、标签页、框架、iframe都是浏览上下文；对JS来说，他们都是window对象。而对于嵌套的上下文来说，其并不相互独立，在窗口中运行的JS程序可以通过API访问到祖先或子窗口，但会受到同源策略的限制。每个窗口或iframe等都有独立的JS执行上下文。

打开或关闭窗口

window.open()：

此全局对象方法可以打开一个新的窗口或标签页。载入指定的URL到新的窗口或已存在的窗口，返回那个窗口的window对象。

有四个参数:

  1. 要在窗口中显示的文档的URL。此参数省略，则使用空页面的URL => about:blank
  2. 新打开窗口的名字。如果是使用的已存在窗口的名字，则使用已存在的窗口。省略此参数，则使用指定名字”_blank”打开新的未命名窗口。
  3. 一个以逗号分开的列表，包括大小和各种属性，表明新窗口是如何打开的。
  4. 布尔值。第四个参数只有在第二个参数命名的是一个已存在的窗口时才有用。参数为true时，表明打开的指定URL替换掉当前窗口已打开文档的历史记录条目；为false 时，则在当前窗口历史记录中创建一个新的历史条目。后者为默认值。

当前的标签页打开里一个标签页时，两个window对象需要相互通信，新标签页则不能运行在独立的进程中。新创建的窗口或标签页，其有个opner属性，表示打开新窗口的window对象。

如果将opener属性设为null，则表示告诉浏览器新打开的窗口或标签页与打开的它的窗口或标签页之间不需要通信，新窗口可在独立的进程中运行，标签页间的通信一旦切断，则没有办法恢复。

可以使用window.close()方法关闭一个窗口，window可以时其他打开的窗口的名字。即使一个窗口关闭了，代表它的window对象仍然存在，已关闭的窗口会有一个值true的closed属性，其document为null，方法也不再工作。

窗体间引用

新打开的窗口的opener属性指向打开它的窗口，open方法返回新打开的窗口的window对象，这样两个窗口间就可以相互引用对方的方法和属性。

任何窗口或窗体中的JS代码都可以将自己的窗口或窗体引用为window或self。可以用parent属性引用包含窗口或窗体的window对象。如果窗口时顶级窗口，parent属性引用的就是本身。如果一个窗口嵌套较深，要引用顶层窗口，可使用top属性，指向最顶层窗口。

子窗体可以通过<iframe>标签创建。可以通过引用元素的方法引用子窗体。而对子窗体的window对象的引用，可以通过<iframe>元素的contentWindow属性引用window对象。

获得了子窗体的window对象，可以反向引用到窗体的iframe元素。可以通过window对象的frameElement属性引用得到。

一般而言，通常使用window对象的frames属性引用子窗口。frames属性引用的是类数组对象，里面包含窗口含有的子窗口。可以通过数字或窗体名索引。

frames类数组里面的元素是window对象，而不是iframe元素。
